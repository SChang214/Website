Checkers

Software / Java / GUI | 2024

Coded two-player checkers with move-validation.

Coded checkers in Java, then built a GUI primarily using the java.swing library.

Engineering problem

- Two-users must be able to play checkers using one screen

- Due to it's nature as an early career project, I did not have the opportunity to check as many edge-cases as I would have liked

Classes

- Game.java : main java file

- ButtonGrid.java : creates buttons and puts them in a window (GUI)

- CheckersBoard.java : puts the buttons into a checkers format and does styling (GUI)

- Matrix.java : matrix of booleans

- Player.java : Tracks player moves. Checkers logic is implmented here

Contributions

- Built a functional a 2-player checkers game, integrating win-state detection, board visualization, move validation and king promotion logic

- Developed a button based checkers board GUI using java.swing to display the board in an intuitive fashion

- Tracked checker stones and king states and positions using a matrix based system to manage board logic

- Reduced code written by 50\% by implementing object oriented programming

Reflection

If future iterations were made, an effort should be made to clean up code and simplify logic used in coding.
